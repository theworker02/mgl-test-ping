# Contributing to MGL

Thank you for your interest! Please read this guide before opening a pull request.

## Branch Naming
- `feature/` — new features
- `fix/` — bug fixes
- `docs/` — documentation

## Commit Format
```
feat(scope): description
fix(scope): description
docs(scope): description
```

## PR Process
1. Fork the repo
2. Create a branch
3. Make changes with tests
4. Open a PR against `main`
5. One maintainer review required

See the full guide in the repository.
